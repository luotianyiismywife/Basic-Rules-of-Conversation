## ✨ 已实现功能

### 双阶段流程（单文件）
`双人/双人基础规则.md` 包含完整的角色扮演交互流程，按"当前所处阶段 → 收到什么输入 → 输出什么内容"的决策逻辑编排。

**阶段1：初始化**
- 三种模式可选：随机模式 / 指定模式 / 导入模式
- 随机模式：AI 直接生成完整场景（旁白 + 状态 + 记忆）
- 指定模式：用户提供部分或全部场景要素，AI 自动补全缺失要素并生成场景；不满意可用 `设定{}` 微调
- 导入模式：上传之前的对话导出文件继续

**阶段2：正式对话**
- 每轮按标准输出流程执行：解析输入 → 生成叙事（旁白+动作+台词） → 输出记忆/待办/状态
- 状态是叙事的快照，叙事推进到哪里状态就同步更新到哪里

### 信息管理体系
- **记忆{}**：记录历史信息，分有效/归档/无效三层，自动清理防膨胀
- **状态{}**：场景和角色的当前快照，含关联规则（物品间、人物间、人物与场景间）
- **待办{}**：追踪待完成目标，含5W1H要素，到达触发时间自动提醒
- **设定{}**：回溯性修改，角色不自知设定被修改；阶段1跳过叙事，阶段2生成仅环境旁白

### 核心规则
- **叙事标签系统**：旁白{}/（动作描写）/"台词" 三要素每轮必含
- **人称代词禁令**：禁止使用他/她/他们指代角色，必须用角色名
- **空行格式**：不同标签组间以空行分隔，同一组内状态连续输出
- **R18协议**：触发条件→执行→收束 完整流程
- **导出/导入系统**：标准格式的对话导出与还原

## 🗺️ 待实现功能

### 双 Agent 分拆
当前 `双人/双人基础规则.md` 将初始化（模式选择→场景构建→状态初始化）和正式对话揉在一个文件里，用"阶段"做逻辑分割。但由于 AI 容易在阶段间混淆，计划拆分为两个独立的 agent 指令文件：

| 文件 | Agent 用途 | 职责 |
|------|-----------|------|
| `双人/初始化阶段.md` | **初始化 Agent** | 模式选择 → 场景构建 → 状态初始化 → 输出「情景设定完成」后退出 |
| `双人/正式对话阶段.md` | **角色扮演 Agent** | 收到初始化结果后接手，按标准输出流程执行每轮互动，维护记忆/状态/待办 |

**优势：**
- 每个 Agent 只聚焦一个任务，不存在"我当前在哪个阶段"的混淆
- 指令文件各自独立，互不干扰
- 可通过 Agent 切换机制（或脚本）串联两个 Agent
