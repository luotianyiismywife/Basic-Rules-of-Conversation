# 测试用例清单 + 预期结果

对应 `测试用例_example.py` 中的 TC1 ~ TC9。
角色以 `角色卡-样例.md` 中的 **林梦雨（小雨）** 为例。

---

### TC1：模式选择

**输入：**（无，首次对话）

**前置上下文：** 无（AI 首次启动，空历史）

**预期输出：**
```
请选择模式：
[随机模式] - AI直接生成场景，无需用户提供设定
[指定模式] - 用户提供部分或全部设定，AI补充缺失要素后生成场景
[导入模式] - 上传之前的对话导出文件继续
```

**检查项：**
- ✅ 仅输出模式选择模板，无角色台词/动作/状态
- ✅ 模板格式精确匹配三条选项

---

### TC2：指定模式 — AI索要场景要素

**输入：**
```
指定模式
```

**前置上下文：** AI 已输出模式选择模板，用户回复「指定模式」

**预期输出：**
```
请设定场景要素（时间、地点、场景描述等），可只提供部分要素，AI将自动补充其余内容。
```

**检查项：**
- ✅ AI 确认选择后输出场景要素设定提示
- ✅ 无旁白/角色台词/动作/状态标签
- ✅ 提示语明确「可只提供部分要素，AI将自动补充」

### TC3：指定模式 — 用户提交完整场景模板

**输入：**
```
旁白{小雨窝在客厅沙发上，穿着宽大的白T恤和居家短裤，两条腿交叠着，茶几上放着半杯三分糖去冰的奶茶。她正低头专注地玩着Switch，屏幕上亮着星露谷的农场画面，听到玄关传来钥匙声，抬起头看过去}

状态{归属: 场景, 区域: 客厅, 光线: 暖黄灯光, 气温: 温暖, 音量: 轻柔的动漫BGM, 时间: 2026-07-04T16:30:00, 物品: [沙发(小雨窝着)，茶几(半杯奶茶)，Switch(在手上)]}

状态{归属: 小雨, 穿着: 白T恤配居家短裤, 动作: 玩游戏, 位置: 客厅沙发上, 情绪: 悠闲, 体力: 正常}

状态{归属: 晚晴, 穿着: 衬衫配牛仔裤, 动作: 正在开门, 位置: 玄关门口, 情绪: 正常, 体力: 正常}
```

**前置上下文：** 用户已选择指定模式，AI 已输出「请设定场景要素」提示

**预期输出模板：**
```
旁白{（保留用户旁白并可扩写，不删改原文细节）}

状态{归属: 场景, ...（保留用户提供的字段，可补充不可修改）}
状态{归属: 小雨, ...}
状态{归属: 晚晴, ...}

情景设定完成！请发送任意内容开始互动，或使用设定{}/推进{}/导出对话{}进行其他操作。
```

**检查项：**
- ✅ 用户提供的旁白+状态原文保留，不删改关键词（如"三分糖去冰的奶茶""星露谷"）
- ✅ 可以扩写追加内容，不能改写已明确的细节
- ✅ 末尾带「情景设定完成」模板
- ✅ 无记忆{}/待办{}（等待期省略）
- ✅ 无角色动作/台词（等待期禁止）
